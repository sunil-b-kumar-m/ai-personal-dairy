# Auth & RBAC Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add local + OAuth authentication, JWT session management, fully dynamic RBAC, and admin UI for managing users/roles/permissions.

**Architecture:** Passport.js strategies (local, Google, Microsoft) handle auth. JWTs in httpOnly cookies manage sessions. Prisma models (User, Role, Permission with many-to-many joins) power a dynamic RBAC system. Express middleware chain: authenticate → authorize. React frontend with auth context, protected routes, and admin CRUD pages.

**Tech Stack:** Passport.js, bcrypt, jsonwebtoken, cookie-parser, Zod, Prisma, React, Tailwind, react-hot-toast
